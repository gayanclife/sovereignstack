# Development history

These are implementation notes from the phases of work that produced the
1.0 release. They're kept for archaeology — when someone asks "why is
this designed this way?", these docs answer.

If you're a user, you don't need to read any of these. Start at
[`docs/QUICKSTART.md`](../QUICKSTART.md).

If you're a contributor:

- [`PROGRESS_SUMMARY.md`](PROGRESS_SUMMARY.md) — the bird's-eye view
- [`TASKS.md`](TASKS.md) — the original task list, with status
- [`PHASE_1_COMPLETION.md`](PHASE_1_COMPLETION.md), `PHASE_2*`, `PHASE_3` — early-phase write-ups
- [`TESTING_MULTIMODEL.md`](TESTING_MULTIMODEL.md) — manual multi-model test recipe

The Phase A–G docs live in [`docs/`](../) (one level up) because they
correspond to capabilities that ship in the 1.0 binary; these
`*_COMPLETION` files are status snapshots, not user-facing reference.
